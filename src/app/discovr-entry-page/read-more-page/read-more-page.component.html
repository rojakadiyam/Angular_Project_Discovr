Anomaly detection

DiscovR is a novel algorithm to detect anomaly.

Anomaly detection is an important tool for detecting fraud, detecting AML transactions, network intrusion, risk mitigation, manufacturing defects, engine defects, detect serious disease based on iot devices and other rare events that may have great significance for the business but are hard to find.

DiscovR is a novel algorithm to detect anomaly. As it is based on unsupervised machine learning algorithm, it does not require labeled data. It delivers speed, accuracy, and context better than any solution of its kind to find abnormal (or unusual) data, activity, and transactions in any given datasets.

It has been successfully tested on hundreds of public datasets over several years. DiscovR is typically used to determine serious deviations in any given dataset.

Any deviation would indicate one of the following:

  • Errors in Data creation.
  • A positive trend that informs decision makers to better utilize any opportunities.
  • A negative trend preparing organizations to take serious steps for risk mitigation.
It is very critical to investigate each deviation and necessary steps should be taken to welcome if there is any new business opportunity or take sufficient measures to alleviate if any serious problem is found.
An anomaly detection tool has the following requirements:

Accuracy:

Industry leading accuracy across hundreds of datasets

Speed:

Best in class speed of processing achieved. Processed 8.5 million data points in less than 20 seconds on Windows 10 and less than 6 seconds on Ubuntu on an ordinary laptop!

Explainability:

User defined Key column references provides relevant feature, its weightage and value with range of common values and whether is high or low

Categorical Columns:

Handles both numeric and categorical data.

Deterministic:

For same input data, the output is exactly same. Most other algorithms provide different result each time we run.

Real time Streaming Data:

Handles streaming data one row at a time with milliseconds response

Intelligent Sorting:

The rows are sorted in descending order of anomaly score, thus the row with the highest anomaly is shown at the top. For each row, key columns are sorted in descending order according to their weightage. So, the first key column in each row is always the one with the highest weightage
